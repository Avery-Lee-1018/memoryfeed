# Design & UX Principles

이 문서는 `memoryfeed`의 QA/픽스 히스토리를 바탕으로 정리한 실전 가이드입니다.  
신규 기능/수정 시 반드시 이 원칙을 우선 적용합니다.

## 1) 핵심 경험 원칙
- 피드는 “설명 UI”가 아니라 “기억을 다시 떠오르게 하는 경험”을 우선한다.
- 한 동작의 결과는 사용자에게 예측 가능해야 한다. (무반응/애매한 상태 금지)
- 작은 인터랙션도 맥락을 해치지 않게 자연스럽게 연결한다. (갑작스러운 점프, 블랙 정지 화면 금지)

## 2) 피드 동작 원칙
- 일자별 피드는 고정성을 우선한다. (임의 재셔플 최소화)
- 오늘 피드에서만 `오늘은 안볼래요`를 제공한다.
- 카드 액션 이후 결과는 반드시 상태 변화가 있어야 한다.
  - 예: 개선 성공 또는 피드 아웃
  - “아무것도 안 바뀜” 상태 금지
- 콘텐츠 교체/신고 플로우는 명시적 닫기 버튼을 제공한다.

## 3) 정보 구조 원칙
- `Feed`와 `My`는 역할이 명확히 달라야 한다.
  - Feed: 소비/회고
  - My: 소스 관리 + 흔적 확인
- 2뎁스 화면(예: 소스별 피드)은 전역 헤더를 단순화하고 뒤로가기 동선을 명확히 한다.
- 정렬/필터 기준은 화면 상단에서 한 번만 제어한다. (카드별 중복 기준 표기 금지)

## 4) 타이포/카피 원칙
- UI 카피는 기능 설명보다 인지/감정 프레이밍을 우선한다.
- 버튼/라벨은 짧고 명확하게 작성한다.
- 동일 역할 라벨은 화면 전반에서 통일한다. (예: `N번 떠오름`, `N개 메모`)

## 5) 인터랙션 원칙
- 화면 전환 방향/리듬은 일관되게 유지한다.
- 닫기 가능한 레이어(오버레이/모달)는 아래 2개를 모두 제공한다.
  - 바깥 영역 탭으로 닫기
  - 명시적 `닫기(X)` 버튼
- 인트로는 “로그인 이벤트 시 1회 재생”을 기본으로 하며, 새로고침 복원 시 재생하지 않는다.

## 6) 레이아웃/간격 원칙
- 헤더/섹션 간 위계 간격은 명확히 분리한다.
- 칩/태그는 줄바꿈 시 `x/y gap`을 모두 명시해 상하좌우 리듬을 통일한다.
- 모바일 브레이크포인트에서만 세로 스택으로 전환하고, 웹에서는 가로 스택을 유지한다.

## 7) 상태/피드백 원칙
- 성공/실패/부분 실패를 구분해서 피드백한다.
- 네트워크 실패 시 사용자가 즉시 취할 수 있는 다음 행동을 함께 안내한다.
- 긴 작업은 스켈레톤/로딩 상태를 제공하고, 완료 시 결과를 분명하게 알린다.

## 8) 신뢰성 원칙
- 썸네일/제목/설명 중 일부가 실패해도 카드 전체가 붕괴되지 않게 방어한다.
- 대체 콘텐츠가 없을 때는 고장난 카드 상태를 유지하지 말고 제거/빈 상태로 처리한다.
- 중복 노출, 404 랜딩, 깨진 메타데이터는 반복 방지 로직으로 흡수한다.

## 9) 성능/비용 원칙
- 응답 경로는 “빠른 반환 + 백그라운드 정리”를 우선한다.
- 수동 갱신은 쿨다운을 두어 과도한 재크롤링을 막는다.
- 썸네일/수집은 캐시를 적극 활용하고, 무거운 작업은 필요한 시점에만 실행한다.

## 10) 신규 기능 체크리스트 (필수)
- [ ] 액션 후 결과가 명확한가? (무반응/애매함 없음)
- [ ] 모바일/웹 모두 레이아웃이 깨지지 않는가?
- [ ] 닫기 가능한 UI에 명시적 닫기 버튼이 있는가?
- [ ] 실패 시 fallback 또는 피드아웃 등 안전 동작이 있는가?
- [ ] 기존 피드 고정성/일자 규칙을 깨지 않는가?
- [ ] 비용/성능 영향을 완화하는 방어 장치가 있는가?
